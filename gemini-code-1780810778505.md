# 專案啟動指令：Supermarket Tracker (AI 購物記帳與匯率換算 Web App)

## 專案概述
請協助我開發一個具備 AI 影像辨識、支援多幣別匯率即時換算，並使用 Docker 部署於 VPS 的 Web 應用程式。使用者可以邊逛超市邊拍照，系統會自動辨識商品名稱與價格，並統整出外幣與新台幣的總花費。

## 🚀 System Directive: Ultracode Mode & Safety Rules
1.  **啟動 Ultracode 模式**: 你必須提供完整、無省略、高度最佳化、且具備完整邊界條件處理的程式碼。絕對禁止使用「// ...在此加入剩餘程式碼」等 Placeholder。所有模組需具備高內聚、低耦合特性。
2.  **絕對優先的安全指令**: 在提供任何涉及刪除容器 (`docker rm`)、移除檔案、或變更資料夾結構的指令前，你必須先暫停，並主動提醒並引導我完成資料備份。這是一條不可打破的規則。

## 系統核心架構：兩段式視覺辨識 (前端快篩 + 後端 AI 精辨)
1.  **前端 TF.js 快速分類 (Offline-First)**:
    * 載入 `MobileNet` 特徵提取模型。
    * 使用者確認商品後，使用 KNN 訓練影像特徵並序列化至 `localStorage` (key: `supermarket_item_classifier`)。
    * 下次拍照若 KNN 置信度 ≥ 0.7 則直接快取命中；若 < 0.7 則 Fallback 呼叫後端 API。
2.  **後端 AI 精辨 (MiniMax VLM API)**:
    * 使用 MiniMax VLM 模型，注意其回應格式為 `{content: "..."}`。
    * AI 回應需經三層 Fallback 解析：JSON 解析 → Regex 抓取 → 啟發式預設處理。

## 影像處理與資料流傳輸
1.  **EXIF 方向修正**: 解析 JPEG `0x0112` 標籤，解決 iPhone 直立拍照圖片橫躺問題。
2.  **雙端三段式圖片壓縮**:
    * **前端**: Canvas API 轉 base64 (品質 0.8 遞減至 0.5，控制在 500KB 內)。
    * **後端**: 收到 base64 後使用 `sharp` 二次壓縮 (1200px fit inside, jpeg quality 85)，存入 `uploads/items/YYYY/MM/DD/`。

## 核心應用功能與 Workflow 支援
1.  **即時匯率換算**: 串接免費匯率 API，具備 Fallback 機制讀取 SQLite 最後有效匯率。
2.  **購物車 UI**: 支援單筆刪除與清空，底部顯示外幣與 TWD 總價。免登入機制（`localStorage` + IP Fingerprint 綁定 SQLite）。
3.  **Workflow 自動化支援**:
    * 實作標準化的 Webhook / RESTful API 端點。
    * 支援事件觸發廣播（Event Emitter），例如：「新增商品成功」、「總價超過預算警戒線」時，可推送 JSON payload 給外部自動化工作流工具。

## 部署與系統韌性
1.  API 重試與限流 (Exponential Backoff, 雙層 Rate Limit)。
2.  監聽 `SIGTERM` 實作 Graceful Shutdown。
3.  SQLite 資料庫、Docker / Docker Compose 部署、Cloudflare Tunnel (綁定 `sm.yuang093.cc`)、GitHub 版本控制 (`https://github.com/yuang093/super.git`)。
